# agents.md — Meetings (inherits /AGENTS.md)

**Scope:** meeting notes and follow-ups.

## Rules
- One file per meeting episode: `Meeting-<context>-<YYYY-MM-DD>.md`.
- Capture: date, attendees, key points, decisions, action items.
- Decisions → promote to the nearest `decisions.md` (project or system). Don't leave decisions only in meeting notes.
- Action items → a task tracker or `Ops/System/Backlog.md`; don't bury them in meeting files.
- Append targets: existing recurring-meeting note files (add new episode as a dated section).
